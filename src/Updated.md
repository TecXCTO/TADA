To simultaneously handle whole-file classification and internal part segmentation, your Python modules must parse topological faces.While .stl and .obj files only contain raw flat triangles (making it incredibly difficult to find true circular holes or internal threading), .step or .iges files contain Boundary Representation (B-Rep) mathematical curves. By using the open-source pythonocc-core library (or a robust mesh topology analyzer like trimesh with discrete feature splitting), you can programmatically extract true geometries.Below is the production-grade implementation for your src/ modules. It extracts sub-features (holes, diameters, threads), automatically renders multi-angle 2D image snapshots of your 3D assets for computer vision models, runs the assembly segmentation loop, and flattens everything into a final CSV table.
